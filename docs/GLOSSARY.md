Показва състоянието на файловете в по-кратък и компактен формат.

### Къде се среща в проекта

Пример:

```powershell
git status --short
```

## `git diff`

### Какво е

`git diff` е команда, която показва разликите между текущите файлове и последното записано Git състояние.

### Защо го използваме

Използваме я, за да видим точно какво е променено преди да направим commit.

### Къде се среща в проекта

Пример:

```powershell
git --no-pager diff -- app/page.tsx
```

---

## `--no-pager`

### Какво е

`--no-pager` казва на Git да покаже резултата директно в терминала, без да отваря pager режим.

### Защо го използваме

Използваме го, за да не се налага да излизаме от pager режим с `q`.

### Къде се среща в проекта

Пример:

```powershell
git --no-pager diff -- app/layout.tsx
```

---

## Pager mode

### Какво е

Pager mode е режим, в който Git показва дълъг резултат страница по страница.

### Защо го използваме

Той е полезен при много дълъг output, но понякога обърква, защото терминалът изглежда сякаш е „заседнал“.

### Къде се среща в проекта

Появи се при първото изпълнение на:

```powershell
git diff -- app/layout.tsx
```

### Бележка

Излиза се от pager mode с клавиша:

```txt
q
```

## Untracked file

### Какво е

### Защо го използваме

Преди commit трябва да решим дали тези файлове да бъдат добавени към Git.

### Къде се среща в проекта

Пример:

?? docs/

## Line endings

### Какво е

Line endings са символите, с които текстовият файл отбелязва края на ред.

### Защо го използваме

Различните операционни системи могат да използват различни line endings, което понякога води до Git предупреждения.

### Къде се среща в проекта

Git показа предупреждение:

```txt
LF will be replaced by CRLF the next time Git touches it
```

---

## LF

### Какво е

LF означава Line Feed.

### Защо го използваме

Това е тип край на ред, използван често в Linux, macOS и много code формати.

### Къде се среща в проекта

Git го спомена в предупреждението за line endings.

---

## CRLF

### Какво е

CRLF означава Carriage Return + Line Feed.

### Защо го използваме

Това е тип край на ред, използван традиционно в Windows.

### Къде се среща в проекта

Git предупреди, че някои LF редове може да бъдат заменени с CRLF.

---

## Newline at end of file

### Какво е

Newline at end of file означава, че текстовият файл завършва с нов ред.

### Защо го използваме

Това е добра практика при code и текстови файлове, защото помага на инструментите да обработват файловете по-чисто.

### Къде се среща в проекта

Беше добавен newline в края на:

```txt
app/layout.tsx
app/page.tsx
```
